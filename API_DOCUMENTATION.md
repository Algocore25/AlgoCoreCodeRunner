# 🚀 AlgoCore Code Runner API Documentation

Welcome to the official API documentation for **AlgoCore Code Runner**, a high-performance, container-ready execution engine for multiple programming languages, coupled with powerful AI integration and email services.

## 🌐 Infrastructure Details
- **Base URL**: `http://localhost:3000` (Local) / `https://<your-deployment-url>` (Production)
- **CORS**: Enabled for all origins (`*`)
- **Rate Limiting**: Sequential execution model (MAX_CONCURRENT=1) to ensure stability on lightweight containers.

### 2. Code Execution

#### **Run Code**
Executes source code in a sandboxed environment and returns the output.
- **URL**: `/run`
- **Method**: `POST`
- **Body Parameters**:
  | Parameter | Type | Required | Description |
  | :--- | :--- | :--- | :--- |
  | `language` | `Number/String` | Yes | Language ID (1-7) or name (e.g., "python") |
  | `sourceCode` | `String` | Yes | The actual code to execute |
  | `input` | `String` | No | Standard input (STDIN) for the program |

- **Sample Request**:
```json
{
  "language": 4,
  "sourceCode": "name = input()\nprint(f'Hello, {name}!')",
  "input": "AlgoCore"
}
```

- **Success Response**: `200 OK`
```json
{
  "output": "Hello, AlgoCore!\n",
  "error": null,
  "exitCode": 0,
  "cpuTime": 45,
  "memory": 2048,
  "timeout": false,
  "compileTime": null
}
```

### 3. Email Service

#### **Send Email**
Sends an email using a pooled transporter with automatic failover support.
- **URL**: `/send-email`
- **Method**: `POST`
- **Body Parameters**:
  | Parameter | Type | Required | Description |
  | :--- | :--- | :--- | :--- |
  | `to` | `String` | Yes | Recipient email address |
  | `subject` | `String` | Yes | Subject of the email |
  | `text` | `String` | Yes* | Plain text content |
  | `html` | `String` | Yes* | HTML content (Either text or html is required) |

- **Sample Request**:
```json
{
  "to": "user@example.com",
  "subject": "AlgoCore Alert",
  "html": "<h1>Success!</h1><p>Your code execution is complete.</p>"
}
```

## ⚠️ Error Handling
All endpoints follow standard HTTP status codes:
- `400 Bad Request`: Missing required fields or invalid input.
- `404 Not Found`: Endpoint does not exist.
- `500 Internal Server Error`: Execution crash or external service failure.

**Error Response Body**:
```json
{
  "success": false,
  "error": "Detailed error message here"
}
```
